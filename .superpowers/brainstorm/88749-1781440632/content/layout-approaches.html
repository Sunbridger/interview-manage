已有基础

  • Tailwind CSS v4 响应式工具类
  • 卡片/表单组件已基本组件化
  • 颜色/间距系统较统一

核心缺陷

  • Sidebar 固定 224px，无折叠/隐藏
  • 所有页面 max-w-* 限制死宽度
  • 无移动端导航（汉堡菜单）
  • 大屏内容区没有利用额外空间

方案对比

A

响应式 Sidebar + 宽内容

移动端 Sidebar 折叠为汉堡菜单；桌面端 Sidebar 可折叠/展开；内容区移除 max-w 限制，改用 grid 自动填充

改动量：中 | 保持现有设计语言

B

顶部导航 + 全宽布局

Sidebar 改为顶部导航栏；移动端顶部导航折叠；内容区全宽，大屏利用多列布局

改动量：大 | 导航模式完全改变

C

混合式：折叠 Sidebar + 内容区分栏

桌面端 Sidebar 默认折叠为图标模式（hover 展开）；内容区根据页面类型采用不同布局（仪表盘多卡片、题库列表+详情分栏）

改动量：中-大 | 体验最好，更现代

💡 推荐方案 C：兼顾桌面大屏利用率和移动端体验，Sidebar 折叠为图标是主流后台做法（Notion、Linear、Vercel），内容区分栏能真正利用大屏空间。
